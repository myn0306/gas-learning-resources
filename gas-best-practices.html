基本編（まずはここから）

GAS開発の基礎となる重要な10項目

1
名前はわかりやすく
必須
関数や変数は「何をする/何が入るか」が分かる名前に。
コードの可読性と保守性が大幅に向上します。
良い例: function sendEmailNotification() const userEmailAddress = "user@example.com" 悪い例: function func1() const a = "user@example.com"
2
コメントで説明を書く
必須
「なぜこの処理をしているのか」を残す。
将来の自分や他の開発者が理解しやすくなります。
// フォーム送信時に重複チェックを行う // 同じメールアドレスでの複数回答を防ぐため function checkDuplicateSubmission()
3
エラー対策を入れる
必須
try/catchで万一失敗しても止まらず、原因を残せるようにする。
本番環境での予期しないエラーを防げます。
try { // 処理内容 } catch (error) { console.log('エラー:', error.toString()); // ログシートに記録など }
4
大事な情報は金庫に保存
必須
APIキーやWebhook URLはプロパティサービスに保存。
セキュリティリスクを大幅に軽減できます。
// 設定 PropertiesService.getScriptProperties() .setProperty('API_KEY', 'your-key'); // 取得 const apiKey = PropertiesService .getScriptProperties().getProperty('API_KEY');
5
処理は小さく分担する
推奨
1つの関数＝1つの役割に分ける。
デバッグが容易になり、再利用性も向上します。
function processFormSubmission() { const data = getFormData(); validateData(data); saveToSheet(data); sendNotification(data); }
6
無駄を減らす（まとめて処理）
必須
getValues/setValuesで一括操作。
実行時間を大幅に短縮できます。
良い例: const values = sheet.getRange('A1:C100').getValues();
悪い例: for (let i = 1; i <= 100; i++) { sheet.getRange(i, 1).getValue(); }
7
マジックナンバー禁止
推奨
定数にして「意味」を名前で表す。
コードの意図が明確になり、変更も容易になります。
良い例: const EMAIL_COLUMN = 2; const MAX_RETRY_COUNT = 3;
悪い例: if (retryCount > 3) { ... } sheet.getRange(1, 2).getValue();
8
シート名や範囲を直書きしない
推奨
定数やプロパティにまとめる。
シート構成の変更に柔軟に対応できます。
const SHEET_NAMES = { RESPONSES: 'フォーム回答', LOG: 'ログ' }; const sheet = ss.getSheetByName(SHEET_NAMES.RESPONSES);
9
データが空でも動くように
必須
if (!data) return; で例外処理。
予期しないエラーを防ぎ、安定性が向上します。
function processData(data) { if (!data || data.length === 0) { console.log('データが空です'); return; } // 処理続行 }
10
テスト関数を作る
必須
本番トリガー前にtest_〇〇()で試す。
本番環境での問題を事前に発見できます。
function test_sendNotification() { const testData = { email: 'test@example.com', name: 'テストユーザー' }; sendNotification(testData); }

応用編（慣れてきたら）

より高度な開発技術とベストプラクティス

11
グローバル変数を乱用しない
推奨
どこからでも触れる変数は混乱の元。
関数に渡して使い、スコープを明確にします。
良い例: function processSheet(sheetName, config) { // 引数として受け取る }
悪い例: var globalSheet; // グローバル変数 function processSheet() { ... }
12
実行時間制限を意識する
必須
6分ルールを意識し、大量処理は分割する。
タイムアウトエラーを防げます。
function processBatch(startRow, batchSize = 100) { const endRow = Math.min(startRow + batchSize, lastRow); // バッチ処理 if (endRow < lastRow) { // 次のバッチをトリガー設定 } }
13
トリガー設計を工夫する
推奨
フォーム送信、時間ベースなど適切に使い分ける。
効率的で安定した自動化が実現できます。
// フォーム送信時 ScriptApp.newTrigger('onFormSubmit') .onFormSubmit().create(); // 定期実行（毎日9時） ScriptApp.newTrigger('dailyReport') .timeBased().everyDays(1) .atHour(9).create();
14
ログを残す仕組みを持つ
推奨
console.logだけでなく「ログシート」を作成。
運用時の問題調査が格段に楽になります。
function writeLog(level, message, data = null) { const logSheet = ss.getSheetByName('ログ'); logSheet.appendRow([ new Date(), level, message, JSON.stringify(data) ]); }
15
例外ケースを想定して書く
必須
未入力やシートが存在しない場合を考慮。
あらゆる状況で安定動作します。
function getSheet(name) { const sheet = ss.getSheetByName(name); if (!sheet) { throw new Error(`シート「${name}」が見つかりません`); } return sheet; }
16
バージョン管理をする
任意
claspやGitHubでコードを管理。
変更履歴の追跡と復元が可能になります。
# clasp使用例 npm install -g @google/clasp clasp login clasp create "プロジェクト名" clasp push # アップロード clasp pull # ダウンロード
17
公開範囲を適切に制御
必須
Webアプリは「誰でも」公開せず必要最小限に。
セキュリティリスクを最小化できます。
推奨設定: • 実行者: 自分 • アクセス: 組織内のユーザー • または特定のユーザーのみ 避けるべき: • アクセス: 全員（匿名ユーザーを含む）

18
自然文でAIに伝える
推奨
やりたいことを具体的に自然文で説明。
AIがより適切なコードを生成してくれます。
良い依頼例: 「Googleフォームの回答を自動でスプレッドシートに保存し、 新しい回答があったらSlackの#通知チャンネルに 『新しい申込みがありました: [名前]さん』 というメッセージを送信したい」 その他のコツ: • エラー文をそのままコピペして解決依頼 • 小さいステップで段階的に進める • 「効率化できますか？」で改善提案を求める • テストデータ作成もAIに依頼可能
効率的で安全なGAS開発により、より良い自動化システムを構築しましょう！
